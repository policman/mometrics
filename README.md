# Mometrics: Система Мониторинга Доступности (Uptime Monitoring)

Mometrics — это полноценное Full-Stack SaaS-решение для непрерывного отслеживания доступности и задержки внешних веб-сервисов и API. Система построена на микросервисной архитектуре и полностью контейнеризирована для обеспечения масштабируемости и простоты развертывания.

## Ключевые Возможности

* ***Непрерывный Мониторинг***: Регулярные асинхронные проверки (*Health Checks*) внешних URL для отслеживания доступности и среднего времени ответа (*Latency*).
* ***Аутентификация***: Безопасная регистрация пользователей и аутентификация через **JWT** (JSON Web Tokens).
* ***Интерактивные Дашборды***: Визуализация исторических метрик доступности (*Uptime Percentage*) и производительности на стороне клиента.

## Технологический Стек

## CI/CD и Инфраструктура (*DevOps / Automation*)

Проект полностью готов к продакшену и развернут с использованием автоматизированных процессов.

### 1. Архитектура Развертывания
* **Контейнеризация**: Использование **Docker Compose** для оркестрации **6 сервисов** (*Backend, Frontend, DB, Redis, Celery Worker, Celery Beat*).
* **Оптимизация**: Применен *Multistage Build* для образа Frontend для минимизации размера и времени развертывания.
* **HTTPS**: Настроен **Nginx** как реверс-прокси и обеспечено сквозное шифрование (*HTTPS*) с использованием **Certbot (Let's Encrypt)** и автоматическим продлением сертификатов.

### 2. Автоматизация Деплоя (CI/CD)
Процесс обновления полностью автоматизирован с помощью **GitHub Actions**. После каждого `git push` в ветку `main` запускается пайплайн, который выполняет следующие шаги на боевом сервере:
1.  Устанавливает SSH-соединение через специальные ключи деплоя.
2.  Выполняет `git pull` для получения актуального кода.
3.  Запускает `docker compose up --build` для пересборки и перезапуска всех контейнеров.
4.  Применяет миграции базы данных (`alembic upgrade head`).

> ***Результат:*** *Процесс обновления продакшен-версии сведен к одному действию: **`git push`***.
